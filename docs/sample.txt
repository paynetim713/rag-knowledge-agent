本地 RAG 知识库示例文档

第一节: 什么是 RAG
RAG 全称 Retrieval Augmented Generation, 检索增强生成. 它的核心思想是让大语言模型在回答问题前, 先从外部知识库里检索出相关材料, 再把材料拼到 Prompt 里, 让模型基于材料回答, 而不是只靠它训练时见过的知识.

RAG 主要解决三个问题:
1. 大模型训练数据有截止日期, 无法获取最新信息.
2. 大模型对私有数据 (公司文档, 个人笔记) 一无所知.
3. 大模型回答时容易产生幻觉, 给出看似合理但事实错误的答案. 把原始材料喂进去, 模型有据可依.

第二节: RAG 系统的核心组件
一个标准 RAG 系统包含 5 个核心组件:
1. 文档加载器 Loader: 把 PDF, TXT, Markdown 等格式的源文件读进来.
2. 文本分块器 Splitter: 把长文档切成小块, 每块的长度适合送进嵌入模型.
3. 嵌入模型 Embedder: 把文本转换成向量 (一串数字), 让语义相似的文本在向量空间里位置接近.
4. 向量数据库 Vector Store: 存储向量, 并支持快速相似度检索.
5. 大语言模型 LLM: 拿到检索结果后, 生成最终答案.

第三节: 本项目的技术选型
本项目用了以下组件: LangChain 作为编排框架, Ollama 提供本地 LLM (Qwen3-8B) 和嵌入模型 (nomic-embed-text), ChromaDB 作为向量数据库. 所有组件都跑在本地, 完全不依赖外部 API, 适合处理敏感数据或离线环境.
